How It Works

The auto-loader pattern automatically registers behavioral hosts and adds is="behavioral-*" attributes to elements with behavior attributes. This gives you the cleanest HTML with minimal setup.

Step 1: Load Behavior

<script src="https://unpkg.com/behavior-fn@0.2.0/dist/cdn/reveal.js"></script>

Behavior auto-registers and stores metadata (observedAttributes, schema).

Step 2: Load Auto-Loader

<script src="https://unpkg.com/behavior-fn@0.2.0/dist/cdn/auto-loader.js"></script>

That's it! Auto-loader automatically:

  • Scans DOM for behavior attributes
  • Registers behavioral hosts with correct observedAttributes
  • Adds is="behavioral-*" attributes
  • Watches for new elements (MutationObserver)

Step 3: Use Clean HTML (No is Attribute Needed)

<!-- Auto-loader adds is="behavioral-reveal" automatically -->
<dialog behavior="reveal" id="my-modal">
  Content here
</dialog>

<button commandfor="my-modal" command="--toggle">
  Open Modal
</button>

The auto-loader automatically detects behavior="reveal", registers the host, and adds is="behavioral-reveal".

Complete Source

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Auto-Loader Pattern Example</title>
  
  <!-- 1. Load behavior -->
  <script src="https://unpkg.com/behavior-fn@0.2.0/dist/cdn/reveal.js"></script>
  
  <!-- 2. Load auto-loader (auto-enables, auto-registers hosts) -->
  <script src="https://unpkg.com/behavior-fn@0.2.0/dist/cdn/auto-loader.js"></script>
</head>
<body>
  <!-- 4. Omit is attributes (auto-loader adds them) -->
  <dialog behavior="reveal" id="my-modal">
    <h2>Hello!</h2>
    <button commandfor="my-modal" command="--hide">Close</button>
  </dialog>
  
  <button commandfor="my-modal" command="--toggle">
    Open Modal
  </button>
</body>
</html>

Tradeoffs

Aspect Explicit Pattern Auto-Loader Pattern
HTML More verbose (is attribute) Cleaner (no is attribute)
Bundle Size Smaller (no auto-loader) +5.4KB
Performance Best (no observer) MutationObserver overhead
Debugging Easy (explicit) Harder (implicit)
Dynamic Content Manual is attributes Automatic detection

Why Auto-Loader?

  • Simplest: Just 2 script tags, no manual setup
  • Clean HTML: No is attributes needed
  • Dynamic content: Auto-detects new elements
  • Quick setup: Perfect for prototypes and demos
