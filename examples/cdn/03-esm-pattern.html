How It Works

Use native ES module imports with type="module". All BehaviorFN bundles have .esm.js versions.

Import and Use

<script type="module">
  // Import core functions
  import { 
    registerBehavior, 
    defineBehavioralHost 
  } from 'https://unpkg.com/behavior-fn@0.2.0/dist/cdn/behavior-fn-core.esm.js';

  // Import behavior factory
  import { revealBehaviorFactory } 
    from 'https://unpkg.com/behavior-fn@0.2.0/dist/cdn/reveal.esm.js';

  // Register behavior
  registerBehavior('reveal', revealBehaviorFactory);

  // Define behavioral host
  defineBehavioralHost('dialog', 'behavioral-reveal');

  console.log('âœ… BehaviorFN loaded via ESM');
</script>

With Import Maps (Recommended)

Use import maps for cleaner imports:

<script type="importmap">
{
  "imports": {
    "behavior-fn/core": "https://unpkg.com/behavior-fn@0.2.0/dist/cdn/behavior-fn-core.esm.js",
    "behavior-fn/reveal": "https://unpkg.com/behavior-fn@0.2.0/dist/cdn/reveal.esm.js",
    "behavior-fn/request": "https://unpkg.com/behavior-fn@0.2.0/dist/cdn/request.esm.js"
  }
}
</script>

<script type="module">
  import { registerBehavior, defineBehavioralHost } from 'behavior-fn/core';
  import { revealBehaviorFactory } from 'behavior-fn/reveal';

  registerBehavior('reveal', revealBehaviorFactory);
  defineBehavioralHost('dialog', 'behavioral-reveal');
</script>

With a Bundler (Vite, Webpack, etc.)

If you're using a bundler, install via npm and import directly:

# Install via npm
npm install behavior-fn

# Import in your code
import { registerBehavior, defineBehavioralHost } from 'behavior-fn/dist/cdn/behavior-fn-core.esm.js';
import { revealBehaviorFactory } from 'behavior-fn/dist/cdn/reveal.esm.js';

registerBehavior('reveal', revealBehaviorFactory);
defineBehavioralHost('dialog', 'behavioral-reveal');

With Auto-Loader (ESM)

You can also use the auto-loader with ESM:

<script type="module">
  import { 
    registerBehavior, 
    defineBehavioralHost 
  } from 'https://unpkg.com/behavior-fn@0.2.0/dist/cdn/behavior-fn-core.esm.js';
  
  import { revealBehaviorFactory } 
    from 'https://unpkg.com/behavior-fn@0.2.0/dist/cdn/reveal.esm.js';
  
  import { enableAutoLoader } 
    from 'https://unpkg.com/behavior-fn@0.2.0/dist/cdn/auto-loader.esm.js';

  // Register behavior
  registerBehavior('reveal', revealBehaviorFactory);

  // Enable auto-loader
  enableAutoLoader();

  console.log('âœ… BehaviorFN loaded with auto-loader');
</script>

<!-- Now you can omit is attributes -->
<dialog behavior="reveal" id="my-modal">
  Content
</dialog>

Browser Support

  • ES Modules: All modern browsers (Chrome 61+, Firefox 60+, Safari 11+, Edge 79+)
  • Import Maps: Chrome 89+, Firefox 108+, Safari 16.4+, Edge 89+
  • Fallback: Use the IIFE bundles (.js instead of .esm.js) for older browsers

Benefits of ESM Pattern

  • Modern: Uses standard ES module syntax
  • Tree-Shakeable: Bundlers can remove unused code
  • Type-Safe: Works with TypeScript out of the box
  • No Globals: Doesn't pollute window namespace
  • Scoped: Module scope prevents naming conflicts

Other Patterns

  • Explicit Pattern - Recommended for production
  • Auto-Loader Pattern - Convenience for prototypes
