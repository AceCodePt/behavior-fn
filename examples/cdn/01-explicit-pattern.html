How It Works

The manual pattern gives you full control. You load the behavior and manually define the behavioral host.

Step 1: Load Behavior

<script src="https://unpkg.com/behavior-fn@0.2.0/dist/cdn/reveal.js"></script>

Behavior auto-registers and stores metadata in window.BehaviorFN.behaviorMetadata.

Step 2: Define Behavioral Host

<script>
  // Get metadata (includes observedAttributes)
  const meta = BehaviorFN.behaviorMetadata['reveal'];
  
  // Define the behavioral host for this tag + behavior combo
  BehaviorFN.defineBehavioralHost('dialog', 'behavioral-reveal', meta.observedAttributes);
</script>

This tells the browser how to upgrade <dialog is="behavioral-reveal"> elements.

Step 3: Use Explicit is Attributes

<dialog is="behavioral-reveal" behavior="reveal" id="my-modal">
  Content here
</dialog>

<button commandfor="my-modal" command="--toggle">
  Open Modal
</button>

The is="behavioral-reveal" attribute tells the browser to use the behavioral host you defined.

Complete Source

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Manual Pattern Example</title>
  
  <!-- 1. Load behavior -->
  <script src="https://unpkg.com/behavior-fn@0.2.0/dist/cdn/reveal.js"></script>
  
  <!-- 2. Define behavioral host -->
  <script>
    const meta = BehaviorFN.behaviorMetadata['reveal'];
    BehaviorFN.defineBehavioralHost('dialog', 'behavioral-reveal', meta.observedAttributes);
  </script>
</head>
<body>
  <!-- 3. Use explicit is attributes -->
  <dialog is="behavioral-reveal" behavior="reveal" id="my-modal">
    <h2>Hello!</h2>
    <button commandfor="my-modal" command="--hide">Close</button>
  </dialog>
  
  <button commandfor="my-modal" command="--toggle">
    Open Modal
  </button>
</body>
</html>

Why Manual Pattern?

  • Performance: Smallest bundle (no auto-loader)
  • Control: Explicit host definition
  • Debugging: Easy to see what's applied where
  • Standard: Uses standard Web Components API
  • Size: 8.7KB minified (3.2KB gzipped)

Other Patterns

  • Auto-Loader Pattern - Simpler (recommended for most)
  • ESM Pattern - For bundlers
