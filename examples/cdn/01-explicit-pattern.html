How It Works

The explicit pattern uses the is attribute to tell the browser which custom element to use. No JavaScript magic, no auto-loader, just standard Web Components.

Step 1: Load Core Runtime

<script src="https://unpkg.com/behavior-fn@0.2.0/dist/cdn/behavior-fn-core.js"></script>

The core runtime (~4KB) provides the foundation: behavior registry, host management, and utilities.

Step 2: Load Behaviors You Need

<script src="https://unpkg.com/behavior-fn@0.2.0/dist/cdn/reveal.js"></script>

Each behavior auto-registers when loaded. Load only what you need!

Step 3: Use Explicit is Attributes

<dialog is="behavioral-reveal" behavior="reveal" id="my-modal">
  Content here
</dialog>

<button commandfor="my-modal" command="--toggle">
  Open Modal
</button>

The is="behavioral-reveal" attribute tells the browser to upgrade the element. The behavior="reveal" attribute tells which behavior to apply.

Complete Source

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Explicit Pattern Example</title>
  
  <!-- 1. Load core runtime -->
  <script src="https://unpkg.com/behavior-fn@0.2.0/dist/cdn/behavior-fn-core.js"></script>
  
  <!-- 2. Load behaviors -->
  <script src="https://unpkg.com/behavior-fn@0.2.0/dist/cdn/reveal.js"></script>
  <script src="https://unpkg.com/behavior-fn@0.2.0/dist/cdn/logger.js"></script>
</head>
<body>
  <!-- 3. Use explicit is attributes -->
  <dialog is="behavioral-reveal" behavior="reveal" id="my-modal">
    <h2>Hello!</h2>
    <button commandfor="my-modal" command="--hide">Close</button>
  </dialog>
  
  <button commandfor="my-modal" command="--toggle">
    Open Modal
  </button>
</body>
</html>

Why Explicit?

  • Performance: No MutationObserver watching the DOM
  • Predictability: Element behavior is visible in HTML
  • Debugging: Easy to see what's applied where
  • Standard: Uses standard Web Components API
  • Simple: No magic, no surprises

Other Patterns

  • Auto-Loader Pattern - Convenience for prototypes
  • ESM Pattern - For bundlers
